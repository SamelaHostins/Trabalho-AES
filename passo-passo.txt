1- Configuração Inicial:
- EU como usuario
  DEVO informar o arquivo a ser criptografado. 
  DEVO informar o nome do arquivo de destino a ser gerado.
  DEVO fornecer a chave de criptografia
- A chave de criptografia deve ter 128 bits (16 bytes).
- Deve ser um campo texto em que possa ser fornecido os valores dos 

2- Validar as entradas do usuário para fazer testes e não correr o risco de errar
- DEVE ser validado se o caminho do arquivo é válido
- DEVE ser validado se não foi passado um campo vazio ou nulo
- DEVE ser validado se o nome do arquivo de destino não é nulo ou vazio
- DEVE ser validado se não foi passado uma chave vazia ou nula
- DEVE ser validado se foi passada uma chave em decimal, separando-os por vírgula
- DEVE ser validado se foi passada uma chave de 16 bytes
- DEVE ser validado se os decimais passados na chave são <= 255

3- Leitura do Arquivo de Entrada:
- Ler o arquivo de entrada em blocos de 128 bits (16 bytes). 
- Certificar de aplicar o preenchimento PKCS#7, 
- Se necessário, preencher o último bloco se ele não tiver 16 bytes completos.

4- Expansão da Chave:
- Expanda a chave de 128 bits para gerar as subchaves para cada rodada do AES. 
- No modo ECB, as subchaves não mudam durante a cifragem de diferentes blocos.

5- Cifragem dos Blocos:
Para cada bloco de 128 bits:
- Aplicar a operação SubBytes ao bloco.
- Aplicar a operação ShiftRows ao bloco.
- Aplicar a operação MixColumns ao bloco.
- Aplicar a operação AddRoundKey usando a subchave da rodada.

6- Gravação do Texto Cifrado:
- Conforme a cifra cada bloco de dados, gravar o texto cifrado em 
um novo arquivo de saída. 
- Certificar de manter a ordem dos blocos correta.

7- Fim da Cifragem:
- Após cifrar todos os blocos de dados do arquivo de entrada, informar o usuário 
- Que o arquivo está criptografado em seu arquivo de saída.
