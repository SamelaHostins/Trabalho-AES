2- Validar as entradas do usuário para fazer testes e não correr o risco de errar  FEITO
- DEVE ser validado se o caminho do arquivo é válido
- DEVE ser validado se não foi passado um campo vazio ou nulo
- DEVE ser validado se o nome do arquivo de destino não é nulo ou vazio
- DEVE ser validado se não foi passado uma chave vazia ou nula
- DEVE ser validado se foi passada uma chave em decimal, separando-os por vírgula
- DEVE ser validado se foi passada uma chave de 16 bytes, não pode ter a mais, nem a menos
- DEVE ser validado se os decimais passados na chave são <= 255

3- Leitura do Arquivo de Entrada:  FEITO
- Ler o arquivo de entrada em blocos de 128 bits (16 bytes). 
- Dividir em blocos de 16 bytes (128 bits)
- Aplicar o preenchimento PKCS#7 apenas no último bloco para garantir que ele tenha o tamanho correto
- Se necessário, preencher o último bloco se ele não tiver 16 bytes completos.

4- Organizar para pós a aplicação do ecb com PKCS#7  FEITO
- Os blocos devem ser organizados em matrizes 4x4
- A chave deve ser organizada em uma matriz de 4x4

5- Iniciar o processo de Expansão de Chave
Implementar o processo de expansão da chave para gerar as subchaves para cada rodada.

6- Cifragem dos Blocos:
Para cada bloco, realizar as etapas do AES no modo ECB:
- Adição da chave (Key Addition) usando a subchave da rodada atual.
- Substituição de Bytes (SubBytes).
- Permutação de Linhas (ShiftRows).
- Mistura de Colunas (MixColumns) (não necessário na última rodada).

7- Gravação do Texto Cifrado:
- Após processar cada bloco, juntar os resultados criptografados.
- Salvar o resultado criptografado em um arquivo de destino fornecido pelo usuário.
